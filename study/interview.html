      # v-if、v-for 优先级

      • 同级渲染的时候，先遍历。判断在循环内执行。循环不可避免，会影响性能。
      • 先判断，再进行遍历。

      # 源码

      • v-if的判断在v-for前面
      if (condition) {}
} else if (el.for && !el.forProcessed) {
return genFor(el, state)
} else if (el.if && !el.ifProcessed) {
return genIf(el, state)
}

      # 结论

      • v-for 优先级更高

      # 优化策略

      • 尽量不要写到一个标签中，否则会每次都先进入循环。
      • v-if 放到外层，使用template进行包裹。
